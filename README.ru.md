## Зачем

Если ты кодишь с AI-агентами (Claude Code, Cursor, Aider) или гоняешь долгие билды, закрытие крышки убивает сессию. Стандартный обход — `pmset disablesleep` — держит Mac бодрым, но **внутренний экран тоже остаётся включённым под закрытой крышкой**, тихо высасывая батарею часами.

Wedge делает две вещи:

1. **Держит Mac бодрым** — переключает `pmset -a disablesleep 1`, и система не уходит в sleep при закрытии крышки.
2. **Гасит внутренний экран при закрытии крышки** — экономит батарею. Открыл крышку — яркость возвращается к той, что была.

## Установка

Скачай свежий `Wedge.zip` со [страницы релизов](https://github.com/wwaannttyy/Wedge/releases), распакуй, перетащи `Wedge.app` в `/Applications/`.

**Первый запуск:** правый клик по иконке → **Open**. Wedge не подписан платным Developer ID, поэтому Gatekeeper ругнётся ровно один раз. Дальше тихо.

Если не хочется кликать через диалог — сними quarantine-атрибут руками:

```bash
xattr -dr com.apple.quarantine /Applications/Wedge.app
```

## Как это работает

Кликни на иконку в менюбаре (мини-ноутбук в профиль) чтобы открыть меню — или **потяни за шнур**:

- **Жми и тяни иконку вниз.** Из менюбара тянется плетёный шнур с бронзовой бусиной-handle, следуя за курсором с физикой натяжения.
- **За порогом drag-а** услышишь "щелчок" и бусина засветится золотым — это коммит.
- **Отпусти** — крышка ноутбука в иконке фиксируется в открытом положении. Клин на месте.
- **Потяни ещё раз** чтобы убрать. Шнур уйдёт обратно, крышка закроется, Mac снова сможет спать.

Пока клин стоит и ты закрываешь крышку — Wedge ставит яркость встроенного дисплея в 0. Откроешь обратно — яркость восстанавливается на ту, что была.

## Возможности

- 🪶 **Лёгкое** — один Swift-бинарь, никаких фоновых демонов и helper-утилит
- 🔒 **Пароль администратора в Связке ключей**, не светится никуда после выхода
- 🌍 **Локализация** — English и Русский
- 🌒 **Автозапуск при входе** через `SMAppService`
- 🧯 **Crash-safe** — если приложение упадёт посередине цикла, яркость восстановится на следующем запуске
- 🎯 **Без приватных демонов, без kext'ов.** Только `pmset` (с паролем из Keychain) и тонкий вызов в `DisplayServices` для яркости

## Настройки

| Настройка | По умолчанию | Заметки |
| --- | --- | --- |
| Запускать Wedge при входе | выкл | Через `SMAppService.mainApp` (один раз подтвердишь в System Settings → Login Items) |
| Гасить экран при закрытии крышки | **вкл** | Главная причина существования Wedge. Можно выключить если нужен чистый `pmset` без яркости. |
| Сохранённый пароль | в Связке ключей | "Забыть" — стирает; Wedge спросит снова при следующей подложке клина |
| Язык | Системный | English / Русский. При смене приложение перезапустится. |

## Требования

- macOS 14 Sonoma и выше
- Желательно Apple Silicon (Intel не тестировался)

## Сборка из исходников

```bash
git clone https://github.com/wwaannttyy/Wedge.git
cd Wedge
./build.sh release
open build/Wedge.app
```

Перегенерировать иконку приложения:

```bash
swift run WedgeIcon
```

## Приватность

- Wedge **никогда не лезет в сеть**.
- Пароль администратора живёт в твоей Связке ключей и доступен только Wedge.
- Wedge ничего не собирает, не передаёт и не логирует.

## Ограничения

- Управление яркостью использует приватный символ из `DisplayServices` — поэтому **в App Store Wedge не попадёт**. Это сознательный размен ради ключевой фичи.
- На Маках без встроенного дисплея (Mac mini, Studio, Pro) шаг с яркостью просто пропускается — остальное работает.
- При очень низком заряде батареи macOS усыпит Mac независимо от `pmset` (это аппаратное поведение, обойти нельзя).
